Quick Overview
Phase 1: Parse, validate, build stack A Phase 2: Measure disorder → route to best sort strategy → execute → clean up

You want clarity on the flow, edge cases, and decision logic before you code.

Phase 1 Refinements
1. Argument Handling
./push_swap          → argc == 1 → return0 silently
./push_swap 42       → single number, already sorted → return 0
./push_swap "5 2 9"→ argc == 2 → ft_split, then validate
./push_swap 5 2 9    → argc > 2 → use argv directly
2. Sign Rules (Clarification)
Standard rule:

Each number can have one leading sign (+ or -).
Signs belong to the number, not between tokens.
Examples: ✅ ./push_swap +42 -17 99 ✅ ./push_swap "-5 +2 9" ❌ ./push_swap 4+2 (sign mid-number) ❌ ./push_swap + 42 (sign separated from digit)

Your validation loop should:

Skip one leading + or - per token.
Ensure the rest are digits.
Reject empty strings after split.
3. INT Overflow & Duplicates
Order matters:

Parse with ft_atoi_long (returns long long).
Check < INT_MIN or > INT_MAX → print "Error\n" to stderr (fd 2), free, exit.
Check duplicates before inserting into stack A.
Pseudo-code: c long long val = ft_atoi_long(token); if (val < INT_MIN || val > INT_MAX) error_exit(stack_a, split_array); if (is_duplicate(stack_a, (int)val)) error_exit(stack_a, split_array); push_to_stack_a((int)val);

4. Flag Parsing
Allowed flags:

--simple, --medium, --complex (force a strategy)
--bench (print metrics at end)
--adaptative (explicit, but it's also the default)
Logic: c int i = 1; while (i < argc && argv[i][0] == '-' && argv[i][1] == '-') { if (!is_valid_flag(argv[i])) error_exit(); // "Error\n" to stderr save_flag(argv[i]); i++; } // Now i points to first number argument

If user mixes flags and numbers weirdly: ./push_swap --simple 5 --bench2→ reject or define clear rule Recommendation: All flags must come first. Once you hit a non-flag, stop flag parsing.

Phase 2 Refinements
1. Disorder Metric
You need a single numeric score that captures "how unsorted is this?"

Options:

Inversion count: Count pairs (i, j) where i < j but stack[i] > stack[j].
LIS-based: n - length_of_longest_increasing_subsequence.
Custom heuristic: Weighted sum of inversions + distance from sorted positions.
Recommendation for adaptive routing:
Use inversion count — it's simple, fast, and correlates well with required moves.

int disorder = count_inversions(stack_a);
if (disorder == 0) {
    free_stack(stack_a);
    return 0; // already sorted
}
2. Strategy Routing
With flag: c if (config.flag == SIMPLE) sort_simple(stack_a, stack_b); else if (config.flag == MEDIUM) sort_medium(stack_a, stack_b); else if (config.flag == COMPLEX) sort_complex(stack_a, stack_b);

Without flag (adaptive): ```c int size = stack_size(stack_a); int disorder = count_inversions(stack_a);

if (size <= 3) sort_simple(stack_a, stack_b); else if (size <= 100|| disorder < size * 0.3) sort_medium(stack_a, stack_b); else sort_complex(stack_a, stack_b); ```

Thresholds are yours to tune — test with random inputs and measure move counts.

3. Bench Output
If --bench was set: c if (config.bench) { printf("Moves: %d\n", move_count); printf("Comparisons: %d\n", comparison_count); printf("Strategy: %s\n", strategy_name); }

Print to stdout (fd 1), after the move sequence.
